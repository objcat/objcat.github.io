```
H哥

关于优先级和开发计划，说一说我的想法，大家讨论下：
1、新功能、新区域的开发优先级可以稍微缓一缓，目前还是尽快更新一版。粉丝等的太久了，已经流失大量活跃玩家了（6月份每天600+日活，现在已经不足300且一直在下降）
2、发版前战斗场景细节稍微优化下。如巨蚁吐子弹、击中雷电、武器还是个薄片等问题。否则即使更新了估计会有不少粉丝会很失望，更新就得不偿失。
3、该版本出来之后的优先级大家可以讨论下，以下几个点：
3.1、真实战斗-人（玩家近期最期待的）：
需要规划至少巨炮阵地内各区域怪物分部、遇敌率、怪物数值、战利品数值等。因为不删档，如果规划不好，轻则玩起来很不爽、重则经济系统就崩了
战斗中的美术、各类不同怪物的攻击方式、人物的攻击方式、效果等
3.2、真实战斗-人车：
3.3、组队战斗
3.4、boss战
4、基于以上，巨炮阵地内完成之后，可以考虑抽出时间优化开发效率。因为各功能点都已经涵盖（大地图、城镇、NPC、交易、战斗、战利品等），后面各区域大多都是重复工作。如果能优化出高效的配置方式，那后续区域的开发将事半功倍。
5、势力玩法、各类活动巴拉巴拉
```

```
鹰哥

大致两块内容：
一个是数据
一个是战斗优化
数据：
1.战斗场景中的数据和非战斗场景同步，比如背包里的物品啥的
非战斗场景和战斗场景的UI是两个
获得的物品、装备、道具、G币等要同步非战斗场景的背包里
2.确定武器装备、怪物的具体数值，
比如弹弓攻击5、手枪20、步枪30、
还有攻击速度，即时步枪攻击力30，但是每秒只能打一发子弹
手枪20，但是美妙2发子弹，实现的伤害其实会比步枪高
武器的数值一旦定下来不能轻易的改，不然玩家会觉得获得的武器没啥意义
数值这块是一个比较大的工作量，得要单独去讨论这个
前期可以先做3-5个武器，后期再慢慢增加
```
