# 一.正文
组件化在iOS开发中已经有了很长一段时间了, 包括微信, qq, 钉钉, 他们也都在使用, 今天接到了一个电话面试, 其中有一个组件化的问题,  怎么解决点击cell跳转到不同类型的页面, 我这里第一个就想到了枚举来设定 比如 1.视频 2.图书 3.广告 跳转到这三种页面, 然后这样会有一个问题, 假如跳转到第四种界面了, 怎么不修改前端来实现这种跳转, 然后我回答了 如果页面没有做好的情况下是怎么都需要修改的, 如果页面做好了就需要修改枚举来达到目的, 这也就动用了前端, 过后又想了想使用组件化的方案确实可以实现这种跳转, 因为组件化使用url来进行页面跳转的比如跳转视频页面 就是 `url://video?参数` 那么我们这套跳转逻辑也可以让后台来传递过来只要带上需要的参数就可以进行这样的跳转了, 这确实是一个方便的地方
